## 🎯 Objective

To consolidate all the concepts learned throughout Week 1 and build a strong foundation in Linux and Git with real-world DevOps relevance.

## 🧠 What I Covered (Week 1 Overview)

### 🔹 Linux Fundamentals
- Command-line navigation (`ls`, `cd`, `pwd`)
- File & directory operations (`mkdir`, `touch`, `cp`, `mv`, `rm`)
- Understanding Linux file system structure

### 🔹 Permissions & Access Control
- File permissions (`chmod`)
- Ownership (`chown`)
- User & group management

### 🔹 System Operations
- Process monitoring (`ps`, `top`)
- Service management (`systemctl`)
- Basic troubleshooting approach

### 🔹 Networking Basics
- Connectivity testing (`ping`, `curl`)
- Port visibility (`ss`, `netstat`)
- Basic debugging of network issues

### 🔹 Text Processing & Logs
- File viewing (`cat`, `less`, `head`, `tail`)
- Searching logs using `grep`
- Understanding `/var/log` and system logs

### 🔹 Package Management
- Installing & managing packages using `apt`
- System updates and cleanup

### 🔹 Bash Scripting
- Variables, conditions (`if`)
- Loops (`for`)
- Writing reusable scripts
- Debugging script errors

### 🔹 Git Basics
- Repository setup (`git init`)
- Tracking changes (`git add`, `git commit`)
- Pushing to remote (`git push`)
- Understanding version control workflow

## ⚠️ Real Errors Faced & Fixes

| Error | Fix |
|------|-----|
| Permission denied | Used `chmod +x` |
| Syntax error in loops | Corrected `for` loop structure |
| File vs directory issue | Fixed path handling |
| Script execution issues | Used proper `./script.sh` |
